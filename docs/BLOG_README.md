## üìã How to Submit a Blog Post

### Step-by-Step Guide

1. **Login as Approved Member**
   - You must be an approved member to submit posts
   - Navigate to the blog page at `/blog`

2. **Click "Submit Your Story"**
   - Look for the blue button in the top-right corner
   - Opens a submission form dialog

3. **Choose Your Category**
   - Select from: Campus Memories, Published Articles, or Talent Hub
   - Read the category description to ensure best fit

4. **Write Your Post**
   - **Title**: Make it catchy and descriptive (max 200 characters)
   - **Excerpt**: Write a compelling summary (max 300 characters)
     - This appears in the blog listing
     - Make it engaging to encourage clicks
   - **Content**: Write your full story
     - Use paragraphs for readability
     - Tell a story with beginning, middle, and end
     - Be authentic and personal
   - **Featured Image**: (Optional) Add a relevant image URL
     - Recommended: 1200px width minimum
     - Direct URL ending in .jpg, .png, etc.
     - Ensure you have rights to use the image

5. **Submit for Review**
   - Click "Submit for Review" button
   - You'll see a success confirmation
   - Your post is now pending admin approval

6. **Wait for Approval**
   - Admins will review your submission
   - Typical review time: 24-48 hours
   - If approved: Your post goes live immediately
   - If rejected: You'll see the reason and can resubmit

## üõ°Ô∏è Admin Review Process

### For Admins Only

1. **Access Admin Dashboard**
   - Navigate to `/admin`
   - Click "Blog Posts" tab

2. **View Pending Submissions**
   - See all pending posts in "Pending" tab
   - Each card shows:
     - Title and excerpt
     - Category badge
     - Author name and department
     - Submission date

3. **Preview Full Content**
   - Click the eye icon to preview
   - Review entire post including:
     - Title, excerpt, content
     - Featured image (if provided)
     - Author information
     - Category appropriateness

4. **Make a Decision**

   **To Approve:**
   - Click "Approve & Publish" button
   - Post immediately becomes visible on the blog
   - Author sees their post published
   - No further action needed

   **To Reject:**
   - Click "Reject" button
   - **IMPORTANT**: Provide a clear rejection reason
   - Examples of good rejection reasons:
     - "Content doesn't fit the selected category. Please submit under 'Campus Memories' instead."
     - "Post contains inappropriate language. Please revise and resubmit."
     - "Excerpt is too long (over 300 characters). Please shorten and resubmit."
     - "Image URL is broken. Please provide a valid image link."
   - Click "Reject Post" to confirm
   - Author will see the reason

   **To Delete:**
   - Click trash icon for permanent deletion
   - Confirm the action
   - Use sparingly - prefer rejection with feedback

5. **Manage Published Posts**
   - View all approved posts in "Approved" tab
   - View all rejected posts in "Rejected" tab
   - Can delete any post at any time

## ‚úçÔ∏è Writing Tips

### Title Best Practices
- Keep it under 200 characters
- Make it descriptive and intriguing

- Include keywords people might search for
- Examples:
  - ‚úÖ "My First Day at Fazlul Huq Hall: A Journey of Discovery"
  - ‚ùå "First Day"
  - ‚úÖ "How I Built a Tech Startup After Graduating from DU"
  - ‚ùå "My Startup"

### Excerpt Best Practices
- Maximum 300 characters
- Summarize the main point
- Create curiosity
- Make readers want to click "Read more"
- Examples:
  - ‚úÖ "Stepping into Fazlul Huq Hall in 1989 changed my life forever. Here's the story of friendships, late-night adda, and lessons that shaped who I am today."
  - ‚ùå "This is about my first day at the hall."

### Content Best Practices
- Use paragraphs (not one big block of text)
- Start with a hook to grab attention
- Tell a story with structure:
  - Beginning: Set the scene
  - Middle: Share the experience/content
  - End: Reflect or conclude
- Be authentic and personal
- Proofread for spelling and grammar
- Use specific details and examples
- Keep it focused on the main topic

- **Recommended Size**: 1200px √ó 630px or similar ratio

  - Imgur: https://imgur.com/
  - Cloudinary: https://cloudinary.com/

## üö® Common Issues & Solutions

### "I can't see the Submit button"
- **Cause**: You're not logged in as an approved member
- **Solution**: 
  1. Make sure you're logged in
  2. Check if your membership is approved
  3. Contact admin if you should be approved but aren't

### "My post isn't showing up"
- **Cause**: It's still pending admin approval
- **Solution**: 
  1. Wait for admin review (24-48 hours typically)
  2. Check admin dashboard if you're an admin
  3. Contact admin for status update

### "Image isn't loading"
- **Cause**: Invalid image URL or broken link
- **Solution**:
  1. Ensure URL ends in .jpg, .png, .gif, etc.
  2. Test URL in a new browser tab
  3. Use a reliable image hosting service
  4. Resubmit with correct URL

### "Excerpt is too long"
- **Cause**: Over 300 character limit
- **Solution**:
  1. Edit to under 300 characters
  2. Focus on the most important information
  3. Save longer explanation for main content

### "Post was rejected"
- **Cause**: Didn't meet submission guidelines
- **Solution**:
  1. Read the rejection reason carefully
  2. Make necessary changes
  3. Submit as a new post
  4. Contact admin if reason is unclear

## üìä Technical Details

### Database: Firestore Collection `blog_posts`
- Status values: `pending`, `approved`, `rejected`
- Published: `true` (visible) or `false` (hidden)
- Categories: `campus-memories`, `published-articles`, `talent-hub`

### API Endpoints
- `POST /api/blog/submit` - Submit new post
- `GET /api/blog/posts` - Get published posts
- `GET /api/blog/posts/[id]` - Get single post
- `GET /api/admin/blog` - Admin: Get all posts
- `PUT /api/admin/blog` - Admin: Approve/reject
- `DELETE /api/admin/blog` - Admin: Delete post

### Security
- Firestore rules enforce all permissions
- Auth tokens validated on every request
- Role-based access control via `user_roles` collection
